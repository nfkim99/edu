2.  데이터 무결성을 위한 제약 조건 4가지는?
	- not null
		: null을 허용하지 않음
	- unique
		: 중복된 값을 허용하지 않고, 항상 유일한 값을 갖도록 함
	- primary key
		: null을 허용하지 않고, 중복된 값도 허용하지않음.
		: not null 조건과 unique 조건을 결합한 형태
		: 중복과 null 허용하지 않는 고유한 값 
	- foreign key
		: 참조되는 테이블의 컬럼의 값이 존재하면 허용


3. 커밋과 롤백에 대하여 설명하시오.

COMMIT (커밋)
	- 모든 작업을 정상적으로 처리하겠다고 확정하는 명령어
	- 사용자가 변경한 데이터의 결과를 저장
	- 트랜젝션의 처리 과정을 데이터베이스에 반영하기 위해서, 
 	  변경된 내용을 모두 영구 저장한다.

ROLLBACK (롤백)

	- 작업 중 문제가 발생했을 때, 트랜젝션 처리과정에서 발생한 변경사항을 취소하고 되돌림.
	- 트랜젝션으로 인한 하나의 묶음 처리가 시작되기 이전의 상태로 되돌린다.
	- TRANSACTION(INSERT, UPDATE, DELETE)작업 내용을 취소한다.
	- 이전 COMMIT한 곳까지만 복구한다.
