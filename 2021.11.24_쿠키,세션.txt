 - 연결성 유지를 위한 꼼수

세션 id 쿠키에 딸려온다.


세션 id

기본적으로 쿠키를 안만들어도,

세션아이디는 클라이언트에 심는다.

서버(톰캣)는 접속자의 구분과 연결성 유지를 위해 쿠키 형태로 심어놓는다.


14강. 세션


웹브라우저와의 관계를 유지시키는 수단 쿠키와 수단이 같다.

단, 클라이언트에 저장하는 쿠키와는 달리 
세션은 서버에서만 접근이 가능하여 보안이 좋고, 저장할 수있는 데이터에 한계가 없다.



쿠키에서는 4kb 300까지만 저장가능
세션은 서버에 저장하기 때문에 보안이 좋고 제한이 없다.

세션은 메모리에 내가 필요한 정보를 올려두고
어느 페이지를 돌아다녀도 써먹겠다는 의미


sessionID : AE9785EB26E48C9D52E0196CEA8A2719
sessionInter : 1800 // 1800초 = 30분

이 세션아이디가 가지는 접속시간 30분

서버마다 sessionInter 시간이 정해져있기 때문에
해당 시간이 지나면 로그아웃됨

세션아이디는 쿠키로도 심고 세션도 심는다.


클라이언트(쿠키세션아이디) <--> 서버 메모리(세션아이디) 와 비교하여

로그인상태를 유지

sessionInter : 1800 // 1800초 = 30분
시간이 지날경우는 세션메모리에서 세션을 지움!!

그래서 로그인 다시하라는 상태가 나옴


웹브라우저마다 세션아이디를 하나씩
